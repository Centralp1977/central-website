# セントラル印刷株式会社 WEBサイト制作ガイドライン

## 1. サイトの目的とターゲット
* 1977年創業の文京区の印刷会社。伝統的な印刷から最先端のXR事業までワンストップで提供。
* メインターゲットは、信頼性を求めるBtoB企業、官公庁、スポーツ、大学などの教育団体、クリエイティブな相談をしたい顧客。

## 2. デザイン・技術ルール
* コーポレートカラー：鮮やかなブルー（`--blue:#0E7AC4`）
* CMYKカラーバー：ヘッダー底辺に3pxのCMYKグラデーション（`#hdr::after`で実装済み）
* 技術スタック：HTML5, CSS3, Vanilla JS（外部ライブラリは極力使わない）
* 必須対応：すべてのコンポーネントはPC・スマホ両対応（レスポンシブ）
* 日本語テキストには `word-break:keep-all` を付与
* 70歳以上の利用者も想定。文字サイズ・コントラスト・タップ領域に注意

## 3. ファイル構成
| ファイル | 役割 |
|---|---|
| `index.html` | トップページ（本番）|
| `central_printing_demo_v5.html` | index.html と常に同内容に保つ（cp でコピー）|
| `company.html` | 会社案内（旧 president.html）|
| `services.html` 等 | 各下層ページ |

## 4. CSS カスタムプロパティ（共通）
```css
--blue:#0E7AC4; --blue-d:#0A5A93; --blue-l:#E8F4FC;
--cyan:#00AEEF; --mag:#EC008C; --yel:#FFF200; --blk:#231F20;
--hh:68px;  /* ヘッダー高さ */
```

## 5. ブランチ・PR ワークフロー（必須）
* **master に直接コミット禁止**
* 作業は必ず `feature/xxx` ブランチ → PR → merge
* PR 作成後は `gh pr merge <番号> --merge --delete-branch` でマージ
* マージ後は新しい feature ブランチで次の作業を開始

## 6. 下層ページ共通ルール
* ヘッダーナビは index.html と同じ完全なナビを使うこと（バックリンクのみ不可）
* スマホ・タブレット（≤900px）はハンバーガーメニューに統一、`nav.desk` は非表示
* セクション見出しを変えるときは、ナビドロップダウン・モバイルメニュー・メタタグも同時更新

## 7. 変更スコープの規律
* 依頼された要素だけを変更する。類似の未依頼要素には手を出さない
* サービス一覧など項目リストを変えるときは、モバイルハンバーガードロップダウンも必ず同時更新
